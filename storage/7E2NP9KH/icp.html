SECTION I

PHP SCRIPT LANGUAGE

Let's start from the quick discuss of what exactly is PHP. PHP (recursive acronym for PHP: Hypertext Preprocessor) is a widely used open source general-purpose scripting language that is especially suited for web development and can be embedded into HTML. PHP can be used on all operating systems and has also support for most of the web servers. So with PHP, you have the freedom of choosing an operating system and a web server. One of the strongest features in PHP is its support for a wide range of databases. Writing a database-enabled web page is incredibly simple using one of the database specific extensions (e.g., for MySQL), or using an abstraction layer like PDO, or connect to any database supporting the Open Database Connection standard via the ODBC extension.

The application described in this article is based on the Tier architecture. This architecture split we can see on the picture below:

Fig. 1 Tier Application Architecture [Introduction: Three-Tiered Web Architecture]

In the three-tier architecture we can distinguish:

  • Data Tier (Back End) — it holds all the data for the web application. In the application we use for that MySQL, an Open Source Database Management System (DBMS) Server
  • Application Tier (Middleware) — also called the logic tier or a business tier. It controls the functionality of the application: logic, decision-making, processing and calculations. For the application it is the Apache HTTP web server.
  • Client Tier (Front End) — called the presentation layer, because it presents the result of the operations caused by our application. The client tier for PHP is usually the web browser.
SECTION II

MYSQL DATABASE

Another very important thing of the application is the MySQL database. This is a very fast, multi-threaded, multi-user, and robust SQL (Structured Query Language) database server. It is used with most web applications based on PHP. A database is a structured collection of data. It may be anything from a simple shopping list to the vast amounts of information in a corporate network. To add, access, and process data stored in a computer database, you need a database management system such as MySQL Server. MySQL is open source which means that it is possible for anyone to use and modify the software. It is very important for people the made their first steps in building web applications. There are many others features of the MySQL, but I would to describe the two distinguish features that implements MySQL and that some other RDBMS systems may not:

  • • Multiple storage engines, allowing one to choose the one that is most effective for each table in the application (Native storage engines - MyISAM, Partner-developed storage engines - InnoDB)
  • Commit grouping, gathering multiple transactions from multiple connections together to increase the number of commits per second.
SECTION III

CMS

We have already discussed the platform on which the system is build and it's the time to discuss the three modules that the application is build of. One of the jobs that I had to do was not only to figure out how to get as many people to visit the site as possible, but to keep them coming back. In need to make the Academic system more attractive. The CMS is a Content Management System. The main feature of the CMS system is that I can easily manage my site content separately from the site design. Armed with the knowledge of PHP and MySQL I was able to create an application to allow users to create new content (authors), edit that content (editors) and publish it. By assigning users to certain roles I could manage who has access to certain functions within the site. The main guidelines that the built CMS is fulfilling are:

  • increasing productivity by having the writers concentrate on writing and the designers on designing
  • allows the editor to review articles and decide which ones should be published
  • presents a consistent look and feel throughout the site using page template
  • • enables the look and feel to be easily changed for a section or throughout the site
  • prevents live content from being changed

SECTION IV

FORUM MODULE

One good way to get users to return to your site is to offer Web forums. These can be used for purposes as varied as philosophical discussion groups and product technical support. Web forums are sometimes also called discussion boards or threaded discussion groups. The idea of a forum is that people can post articles or questions to them, and others can read and reply to their questions. Each topic of discussion in a forum is called a thread. Web forum in the Academic system was build with the following functionality. Users will be able to:

  • start new threads of discussion by posting articles
  • post articles in reply to existing articles
  • view articles that have been posted
  • view the threads of conversation in the forum
  • view the relationship between articles, that is, see which articles are replies to other articles

SECTION V

AUCTION SERVICE MODULE

The last module I want to mention in this article is the auction service module. Everyone who uses Internet knows about such services as Allegro or eBay. Those auction services are nowadays so popular, that some people can't even imagine an afternoon without just checking the product they are bidding. The auction service that was build for my project has got the following functionalities:

  • the service displays a set of different categories for the specified products
  • users with account can add their products for bidding, users without account can only view the products
  • when the auction ends, the owner of the auction and the winner will receive e-mail with the of the auction
  • in the service there will be shown only products which auction time hasn't expired.

As we can see only logged in users can make their own auctions and take part in bidding for products. When the users are adding an auction they can give a description and a photo of the product, and of course the minimal and the maximal bid price. When the auction is over the winner and the owner of the auction will receive e-mail with the summary. Also when there are no biddings or the auction last time is over the similar e-mail will be sent.

The whole action service is pretty simple comparing to Allegro and eBay. The main purpose of that service is to bid small, not valuable products, which can help others for example in studying (scripts, computer programs, books). Building a large and complicated auction service isn't impossible for one person, but it is very hard to achieve.

SECTION VI

SUMMARY

The Academic system was build in web developer technologies to integrate, informate and help students in their everyday life in their student house. By using the three modules: CMS, forum and auction service the Academic system is diversified, so we can get users to return to our site. The whole system was build with PHP and MySQL database. By dividing the application to the three modules, which can be run separately, we can use them for other, more developed systems, or just use as single web site services. By giving the users appropriate privileges to their accounts, they are all the time involved in building, improving and developing the Academic system. In my opinion that will make them often coming back to the site, with enthusiasm and smile on their faces.
